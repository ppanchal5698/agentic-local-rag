# Sprint 04 — TEI Embeddings & Qdrant Hybrid Search

**Source of truth:** `docs/PRD.md`

**PRD references:** 12-Layer Golden Tech Stack (Layers 3, 5), Docker Compose Network Topology Blueprint (`tei-dense`, `tei-sparse`, `qdrant`), FR-1, NFR Security and Privacy

**Depends on:** Sprint 01

**Primary persona:** Data Engineer, AI Developer

---

## Objective

Deploy dense and sparse embedding generation and the Qdrant vector database with native hybrid search and Reciprocal Rank Fusion (RRF), enforcing document-scoped access control at the database level.

---

## Scope

### TEI Dense Embeddings (`tei-dense`)

| Property | Value (from PRD) |
|---|---|
| Docker Image | `text-embeddings-inference` |
| Port Mapping | `8080:80` |
| Volume / State | Model weights cache |
| Hardware | Requires GPU passthrough |

**Responsibilities (from PRD):**

- High-throughput dense semantic embeddings generation.
- Generate 384-dimensional or larger vectors.
- Dynamic batching, scaling across multi-GPU environments.
- Rust-based server optimized for serving dense embedding models at enterprise scale.

### TEI Sparse Embeddings (`tei-sparse`)

| Property | Value (from PRD) |
|---|---|
| Docker Image | `text-embeddings-inference` |
| Port Mapping | `8081:80` |
| Volume / State | Model weights cache |
| Hardware | Requires GPU passthrough |

**Responsibilities (from PRD):**

- Computation of BM25/SPLADE sparse token vectors.
- Compute sparse token indices and weights for BM25 or SPLADE algorithms.

### TEI Load Balancing

- Independent `text-embeddings-inference` Docker containers placed behind an Nginx load balancer or API proxy, distributing encoding requests dynamically across all available GPUs.

### Qdrant (`qdrant`)

| Property | Value (from PRD) |
|---|---|
| Docker Image | `qdrant/qdrant:latest` |
| Port Mapping | `6333:6333`, `6334:6334` |
| Volume / State | `qdrant_data` volume |

**Responsibilities (from PRD):**

- Advanced hybrid vector storage, RRF scoring, and high-speed retrieval.
- Named vectors for the same document payload — dense semantic vectors alongside sparse lexical vectors (BM25 or SPLADE).
- Sparse vectors represented as pairs of non-zero indices and corresponding weights in an inverted index structure separate from the dense HNSW graph.
- Reciprocal Rank Fusion (RRF) executed natively within the database engine.
- gRPC on port `6334` for high-throughput, low-latency sub-requests.

**RRF formula (from PRD):**

$$score(d \in D) = \sum_{r_d \in R(d)} \frac{1}{k + r_d}$$

Where $d$ represents the document, $R(d)$ represents the set of rank positions from the sub-queries, and $k$ acts as a smoothing constant.

### FR-1 — Hybrid Vector Search

| Field | Value (from PRD) |
|---|---|
| **Feature Category** | Hybrid Vector Search |
| **Detailed Requirement** | The system must query both semantic meaning and lexical exact-matches simultaneously. |
| **Acceptance Criteria** | Qdrant successfully executes RRF scoring, blending dense embeddings and sparse BM25 indices in a single API call. |

### NFR — Security and Privacy (database-level enforcement)

- Document-scoped retrieval must be strictly enforced at the database level.
- A user querying the system must only receive vector chunks originating from documents they explicitly own or have Role-Based Access Control (RBAC) clearance to view.
- Qdrant metadata filtering must apply this restriction prior to RRF scoring.
- PostgreSQL manages document metadata and Access Control Lists (ACLs).

## Deliverables

1. `tei-dense` container on port `8080:80` with GPU passthrough and model weights cache.
2. `tei-sparse` container on port `8081:80` with GPU passthrough and model weights cache.
3. Nginx load balancer or API proxy distributing encoding requests across TEI GPU instances.
4. `qdrant` container on ports `6333:6333` and `6334:6334` with `qdrant_data` volume.
5. Named dense and sparse vectors with RRF fusion in a single API call.
6. Qdrant metadata filtering for ACL/RBAC enforcement prior to RRF scoring.

---

## Acceptance Criteria

- [ ] `tei-dense` and `tei-sparse` are deployed with GPU passthrough and model weights cache volumes.
- [ ] Dense embeddings (384-dimensional or larger) and sparse BM25/SPLADE token vectors are generated by independent TEI instances.
- [ ] `qdrant` is deployed on ports `6333:6333` and `6334:6334` with `qdrant_data` volume persistence.
- [ ] **FR-1:** Qdrant successfully executes RRF scoring, blending dense embeddings and sparse BM25 indices in a single API call.
- [ ] Qdrant metadata filtering enforces document-scoped ACL/RBAC restrictions prior to RRF scoring.
- [ ] gRPC retrieval on port `6334` resolves in milliseconds.
