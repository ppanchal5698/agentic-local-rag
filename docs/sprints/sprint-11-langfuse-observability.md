# Sprint 11 — Langfuse Tracing & Observability

**Source of truth:** `docs/PRD.md`

**PRD references:** 12-Layer Golden Tech Stack (Layer 6), Docker Compose Network Topology Blueprint (`langfuse-web`, `langfuse-worker`), FR-5

**Depends on:** Sprint 01, Sprint 10

**Primary persona:** AI Developer, System Administrator

---

## Objective

Deploy self-hosted Langfuse to capture the full trace hierarchy of agentic reasoning paths, token consumption, and execution latencies.

---

## Scope

### Langfuse Platform (from PRD)

- Open-source, LLM-specific observability platform.
- Designed to capture entire trace hierarchy of complex, non-linear agentic reasoning paths.
- Self-hosted via Docker Compose for data sovereignty — ensuring no proprietary internal queries leave the Virtual Private Cloud (VPC).
- Requires three interconnected components:
  1. Langfuse Server API
  2. PostgreSQL database for raw storage
  3. Langfuse Worker container — asynchronously aggregates raw trace data to power real-time dashboards without blocking the main application thread

### `langfuse-web` Service

| Property | Value (from PRD) |
|---|---|
| Docker Image | `langfuse/langfuse:3` |
| Port Mapping | `3000:3000` |
| Volume / State | Stateless (Reads Postgres) |

**Responsibilities (from PRD):**

- Visual dashboard for analyzing traces, token costs, and DeepEval metric overlays.

### `langfuse-worker` Service

| Property | Value (from PRD) |
|---|---|
| Docker Image | `langfuse/langfuse:3` |
| Port Mapping | N/A |
| Volume / State | Stateless (Writes Postgres) |

**Responsibilities (from PRD):**

- Batch processing of incoming telemetry.
- Aggregating spans without blocking the API.

### Trace Capture (from PRD)

- Every LLM request generated by the system becomes a structured trace.
- The application's Python SDK observes the call, capturing input parameters and metadata, and records the output, token consumption, and any generated errors.
- Creates a trace hierarchy of pipeline spans and child steps.
- Maps the relationship between an agent's initial plan, the specific tool it selected, the raw output from the vector database, and the final synthesized response.

### Integration Point (from PRD Section 3.2 — Delivery and Observability)

- The entire execution graph, tool payloads, latencies, and token costs are pushed asynchronously to the Langfuse Worker.

### Integration Point (from PRD Section 6 — `POST /api/v1/ask` Response)

- Response includes a direct URL to the resulting Langfuse observability trace.

### FR-5 — LLM Observability

| Field | Value (from PRD) |
|---|---|
| **Feature Category** | LLM Observability |
| **Detailed Requirement** | Engineering teams must be able to trace full execution paths for debugging. |
| **Acceptance Criteria** | Every query generates a distinct Langfuse trace displaying nested spans (Agent → Tool → Retrieval) alongside precise token costs. |

---

## Deliverables

1. `langfuse-web` container on port `3000:3000` with visual dashboard.
2. `langfuse-worker` container for asynchronous telemetry aggregation.
3. PostgreSQL-backed trace storage (using Sprint 01 PostgreSQL).
4. Python SDK integration capturing structured traces with nested spans.
5. Asynchronous push of execution graph, tool payloads, latencies, and token costs to Langfuse Worker.
6. Langfuse trace URL returned in `POST /api/v1/ask` response payload.

---

## Acceptance Criteria

- [ ] `langfuse-web` is deployed as `langfuse/langfuse:3` on port `3000:3000`.
- [ ] `langfuse-worker` is deployed as `langfuse/langfuse:3` for batch telemetry processing.
- [ ] Langfuse stores traces in PostgreSQL with data sovereignty (self-hosted, no queries leave VPC).
- [ ] Python SDK captures input parameters, metadata, output, token consumption, and errors.
- [ ] **FR-5:** Every query generates a distinct Langfuse trace displaying nested spans (Agent → Tool → Retrieval) alongside precise token costs.
- [ ] Execution graph, tool payloads, latencies, and token costs are pushed asynchronously to Langfuse Worker.
- [ ] `POST /api/v1/ask` response includes a direct URL to the Langfuse observability trace.
- [ ] Dashboard supports DeepEval metric overlays.
